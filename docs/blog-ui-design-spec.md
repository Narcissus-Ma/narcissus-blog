# 博客 UI 设计规范（对齐旧 Hexo 站点）

## 1. 文档信息

- 文档名称：Narcissus 博客 UI 设计规范
- 版本：v1.0
- 日期：2026-03-04
- 视觉基线来源：`https://blog.384385.xyz/`

## 2. 设计目标

1. 保持旧站“科技感 + 卡片化 + 高可读性”的整体印象。
2. 在新技术栈下复刻用户熟悉的交互路径。
3. 保证移动端和桌面端一致体验。

## 3. 旧站视觉基线（必须继承）

### 3.1 主题与色彩 Token

基于旧站 CSS 变量抽取：

#### Light

- 主色：`#425AEF`
- 主色加深：`#1856FB`
- 页面背景：`#F7F9FE`
- 卡片背景：`#FFFFFF`
- 文字主色：`#363636`
- 边框：`#E3E8F7`

#### Dark

- 主强调色：`#F2B94B`
- 页面背景：`#18171D`
- 卡片背景：`#1D1E22`
- 文字主色：`#F7F7FA`
- 边框：`#42444A`

#### 状态/辅助

- 成功：`#57BD6A`
- 警示：`#FFC93E`
- 错误：`#D8213C`（暗色可用 `#FF3842`）

### 3.2 布局基线

1. 顶部导航固定高度：`60px`。
2. 主容器宽度：

- 首页顶区最大宽度：`1400px`。
- 正文布局容器最大宽度：`1200px`。

3. 侧栏宽度：`300px`。
4. 圆角体系：`8px / 12px / 50px`（胶囊按钮）。
5. 阴影体系：轻阴影（默认）+ 主色阴影（hover）。

### 3.3 关键模块结构（首页）

1. 固定导航栏（站点名、菜单、搜索、随机、主题切换）。
2. 顶部内容区（随机 Banner + 分类快捷入口 + 推荐卡片组）。
3. 分类滚动条（Category Bar）。
4. 文章卡片列表（卡片封面 + 文本信息）。
5. 右侧信息卡（作者、标签云、归档、统计）。

### 3.4 关键动效

1. 卡片 hover：边框高亮 + 阴影提升。
2. 封面图 hover：轻微放大（约 1.03）。
3. 导航固定后：背景切换至卡片底色，边框显性化。
4. Banner 遮罩：模糊 + 渐变叠加。

9. 关于页 `/about`
10. 后台 `/admin/*`

## 5. 页面级 UI 规范

### 5.1 首页

- 区块顺序固定：
  1. 固定导航
  2. Hero（随机卡 + 分类入口 + 推荐）
  3. 分类条
  4. 文章列表 + 侧栏
- 文章卡片：
  - 桌面 >1200：双列卡片
  - 平板 768-1200：单列
  - 手机 <=768：封面在上、文案在下

### 5.2 文章详情

- 顶部使用“封面背景 + 标题元信息”模式（继承旧站视觉）。
- 正文区使用卡片容器，保障长文阅读舒适度。
- 目录在桌面端固定，移动端折叠。

### 5.3 列表页（分类/标签/归档）

- 复用首页文章卡片组件。
- 顶部增加面包屑 + 当前筛选条件。

### 5.4 搜索页

- 顶部搜索框 + 结果列表。
- 匹配词高亮。
- 空状态提供“返回首页/查看分类”。

### 5.5 后台管理

- 风格与前台同品牌色，但以效率优先。
- 左侧菜单 + 右侧内容区。
- 表单、表格、抽屉编辑遵循 Ant Design 交互。

## 6. 组件规范

### 6.1 导航栏（`HeaderNav`）

- 高度：`60px`
- 左：站点名/Logo
- 中：菜单（归档、分类、标签）
- 右：随机、搜索、主题切换、返回顶部

### 6.2 分类条（`CategoryBar`）

- 高度：`50px`
- 横向滚动，超出可滑动
- 当前项使用主色胶囊高亮

### 6.3 文章卡片（`PostCard`）

- 结构：封面、标题、摘要、元信息、标签
- Hover：边框高亮 + 阴影增强
- 标题最大 2 行、省略号

### 6.4 侧栏卡片（`SidebarCard`）

- 统一圆角 `12px`
- 内边距 `20-24px`
- 包含：作者卡、标签云、归档、站点信息

### 6.5 Banner 卡（`RandomBanner` / `TodayCard`）

- 保留旧站“视觉吸引 + 引导点击”属性
- 图层：背景图 + 渐变蒙层 + CTA 按钮

## 7. 响应式规范

### 7.1 断点

- `<=768px`：手机
- `769px - 1200px`：平板/小桌面
- `>1200px`：标准桌面

### 7.2 行为规则

1. 手机端：

- 导航压缩为图标优先 + 抽屉菜单。
- Hero 中分类组可折叠或精简。
- 分类条 sticky（顶栏下方）。

2. 平板端：

- 主内容单列，侧栏下移。

3. 桌面端：

- 文章区 + 侧栏并行。

4. 布局实现要求：

- 优先使用 `flex` 完成主布局与组件布局。
- 禁止使用 `float` 作为布局手段。

## 8. 字体与图标

1. 字体优先级：`PingFang SC` / `Microsoft YaHei` / 系统字体。
2. 标题加粗，正文 16px 基准，行高 1.8~2.0。
3. 图标统一使用同一套图标库（与旧站风格一致）。

## 9. 动效与过渡

1. 全局过渡时长：`0.2s ~ 0.35s`。
2. 卡片入场：`slide-in`（低频使用，避免晕动）。
3. 悬浮反馈必须统一：颜色、阴影、边框三者协同。

## 10. 可访问性

1. 正文与背景对比度 >= WCAG AA。
2. 所有交互组件支持键盘焦点样式。
3. 图片必须带 `alt`。
4. 暗色模式下保留足够可读对比。

## 11. 资源与 CDN 建议

1. 静态字体/图标/高亮资源优先使用国内可用 CDN：BootCDN、又拍云。
2. 同时保留本地或第二 CDN 兜底，避免单点故障。
